这个系统是用于分析三种飞机发动机型号（CFM、LEAP、PW）的飞机起降数据

这里面有几个子文件夹，其结构如下:
    --data
        --name1
            --cfm
            --pw
            --leap
            --other docs
        --name2
            --cfm
            --leap
            --pw
            --other docs
        ......
        --name...
            --cfm
            --leap
            --pw
            --other docs
    
    --test_cfm
    --test_leap
    --test_pw
    
    --AGENTS.md

    --OTHER DOCS

文件解释：
    1.对于data文件下的不同子文件夹名字代表了不同人员的数据文件，比如name1就是一个人员的名字，在例如 name1/cfm文件夹中代表了name1人员的CFM发动机飞机操作数据，other docs后面再进行操作这里略去，以此类推到data文件夹中的所有人员，如果某个子文件夹中没有数据，则代表这次分析中该人员无需分析该种机型的数据


    2.对于下方的test_cfm, test_leap, test_pw文件中，CFM、LEAP和PW是三种不同的飞机型号，里面各自的py文件主要是用来分析主目录的datas文件夹里面的数据并生成图像，首先你需要知道CFM、LEAP和PW文件夹中的程序是调用不同类型的csv数据来进行分析，得到某个人的某个机型的飞机操作数据，自动分析怎么运行程序

    对于data文件夹中的人员文件中的数据子文件夹中，csv数据文件的命名规则是文件名包括qlz的是对于CFM飞机，包含qlzlly的是LEAP飞机，包含lly的是PW飞机。注意在之后识别时，qlzlly并不是qlz，lly也并不是qlzlly，也就是说不要错误地把文件名包括qlzlly的数据归类为CFM飞机的数据，要包含且仅包含上述命名规则的时候再识别其飞机类型

你现在要进行的修改和任务如下：
    1.现有的程序调用的是例如CFM、LEAP、PW文件夹内的datas文件夹内的csv文件。需要你将他改为弹出一个可人工选择数据文件夹
    2.现有的代码只能实现单次图像的生成，比如在datas文件夹内全部是CFM飞机的数据，运行一次程序得到一个图,需要你改动以实现以下功能：
        2.1 对选中的数据文件夹中的每个子文件夹进行遍历，比如：选择的是data文件夹，就需要遍历所有人员的文件夹
        2.2 对于每个人员文件夹中的机型文件夹进行遍历，得到同一人员三种不同机型的csv文件，按照文件夹顺序，识别出CFM飞机的数据（即命名中包括qlz），以及上述规则中提到的LEAP飞机的数据（命名中包含qlzlly）以及PW飞机数据（命名包含lly），并将其进行归类，如果没有某个飞机类型的数据则跳过不要卡死，如果在某人员文件夹下面的CFM文件夹中读取到了不是cfm飞机的数据（即命名中不是包括qlz）时，弹窗提示该人员的CFM数据错误，同理应用至其他飞机类型。然后针对不同类型的飞机分别执行对应文件夹中的py程序，比如对于name1文件夹内的所有CFM飞机的数据执行一次CFM飞机的数据分析，对于所有LEAP飞机的数据执行一次LEAP飞机的数据分析，对于所有PW飞机的数据执行一次PW飞机的数据分析，然后按这个规则继续遍历选中的数据文件夹中的其他人员的数据
        2.3 现有的程序运行后生成图的位置和命名需要进行改动，在2.2中的每一次分析后，生成的图片使用“名字_机型”的命名规则，如name1人员的CFM数据分析完成后生成“name1_CFM.png”,以此类推，并存储到主目录下的pics文件夹中，注意不要在之后的图片生成过程中覆盖掉之前的图片
