这个系统是用于分析三种飞机发动机型号（CFM、LEAP、PW）的飞机起降数据

这里面有几个子文件夹，其结构如下:
    --data
        --name1
            --cfm
            --pw
            --leap
            --other docs
        --name2
            --cfm
            --leap
            --pw
            --other docs
        ......
        --name...
            --cfm
            --leap
            --pw
            --other docs
    
    --test_cfm
    --test_leap
    --test_pw
    
    --AGENTS.md

    --OTHER DOCS

文件解释：
    1.对于data文件下的不同子文件夹名字代表了不同人员的数据文件，比如name1就是一个人员的名字，在例如 name1/cfm文件夹中代表了name1人员的CFM发动机飞机操作数据，other docs后面再进行操作这里略去，以此类推到data文件夹中的所有人员，如果某个子文件夹中没有数据，则代表这次分析中该人员无需分析该种机型的数据


    2.对于下方的test_cfm, test_leap, test_pw文件中，CFM、LEAP和PW是三种不同的飞机型号，里面各自的py文件主要是用来分析主目录的datas文件夹里面的数据并生成图像，首先你需要知道CFM、LEAP和PW文件夹中的程序是调用不同类型的csv数据来进行分析，得到某个人的某个机型的飞机操作数据，自动分析怎么运行程序

    对于data文件夹中的人员文件中的数据子文件夹中，csv数据文件的命名规则是文件名包括qlz的是对于CFM飞机，包含qlzlly的是LEAP飞机，包含lly的是PW飞机。注意在之后识别时，qlzlly并不是qlz，lly也并不是qlzlly，也就是说不要错误地把文件名包括qlzlly的数据归类为CFM飞机的数据，要包含且仅包含上述命名规则的时候再识别其飞机类型

你现在要进行的修改和任务如下：
    1.现有的程序调用的是例如CFM、LEAP、PW文件夹内的datas文件夹内的csv文件。需要你将他改为弹出一个可人工选择数据文件夹
    2.现有的代码只能实现单次图像的生成，比如在datas文件夹内全部是CFM飞机的数据，运行一次程序得到一个图,需要你改动以实现以下功能：
        2.1 对选中的数据文件夹中的每个子文件夹进行遍历，比如：选择的是data文件夹，就需要遍历所有人员的文件夹
        2.2 对于每个人员文件夹中的机型文件夹进行遍历，得到同一人员三种不同机型的csv文件，按照文件夹顺序，识别出CFM飞机的数据（即命名中包括qlz），以及上述规则中提到的LEAP飞机的数据（命名中包含qlzlly）以及PW飞机数据（命名包含lly），并将其进行归类，如果没有某个飞机类型的数据则跳过不要卡死，如果在某人员文件夹下面的CFM文件夹中读取到了不是cfm飞机的数据（即命名中不是包括qlz）时，弹窗提示该人员的CFM数据错误，同理应用至其他飞机类型。然后针对不同类型的飞机分别执行对应文件夹中的py程序，比如对于name1文件夹内的所有CFM飞机的数据执行一次CFM飞机的数据分析，对于所有LEAP飞机的数据执行一次LEAP飞机的数据分析，对于所有PW飞机的数据执行一次PW飞机的数据分析，然后按这个规则继续遍历选中的数据文件夹中的其他人员的数据
        2.3 现有的程序运行后生成图的位置和命名需要进行改动，在2.2中的每一次分析后，生成的图片使用“名字_机型”的命名规则，如name1人员的CFM数据分析完成后生成“name1_CFM.png”,以此类推，并存储到主目录下的pics文件夹中，注意不要在之后的图片生成过程中覆盖掉之前的图片


新增结果分析功能：
    在上述所有人员数据分析结束后，新增一个docx文档，按照以下规则依次排列：
        人员名字_CFM
            结果图片
        人员名字_LEAP
            结果图片
        人员名字_PW
            结果图片
        分析：
    先重复一个人的数据，然后按此规则录入所有人的数据到一个docx文档里面，从上到下为不同人的不同机型进行排版

    分析内容如下：

        分析：共选取当月CFM：xxx段，LEAP：xxx段，PW：xxx段，共计xxx段正常航班数据
        1.起飞50ft以下杆量：xxx机型起飞杆量正常/有稳杆，带杆量最大约x个单位，稳杆量最大约y个单位（若有则显示），横侧杆量z个单位
        2.离地姿态：离地曲线正常，xxx机型离地姿态范围
        3.进近1000-100ft杆量：xxx机型带杆量最大x个单位，稳杆量最大y个单位，横侧量最大为z个单位，最大侧风ws节。
        4.落地100ft以下杆量：xxx机型带杆量最大x个单位，稳杆量最大y个单位，横侧量最大为z个单位，最大侧风ws节。
        5.50ft以下曲线及收油门时机：xxx机型收油门高度范围在xxxxx。



        注意：其中的例如xxx，x，y，z等都是具体值，在下面的规则中进行计算指定。对于每个人的不同机型都需要进行分析写入，若某种机型无数据则跳过

    分析规则如下：
        第一行“分析：共选取当月CFM：xxx段，LEAP：xxx段，PW：xxx段，共计xxx段正常航班数据”中，对于上述的xxx段，参考之前读取数据时的结果，如：data文件夹中张三的CFM数据有x条，LEAP数据y条，PW数据z条，则将其带入此处分析，共计段数为x+y+z

        对生成的八个图片进行分析

        1.对于起飞50ft以下杆量：x轴上方代表带杆，x轴下方是稳杆，y轴左侧是左坡度，y轴右侧是右坡度。针对该人的三种机型生成的图片进行分析（若某种机型没有数据则跳过）
            1.1 判断是否有数据位于x轴下方，若无，显示“xxx机型起飞杆量正常”，若有且最下方的数据点处于y轴0~-5个单位以内时显示“xxx机型起飞有少量稳杆”，若超过-5，则显示“xxx机型起飞有稳杆”。稳杆量y为此时最下方数据的纵坐标绝对值，带杆量x为最上方数据的纵坐标值，横侧杆量z是所有数据的横坐标绝对值的最大值。
        2.离地姿态：默认显示离地曲线正常，CFM和PW机型显示数据点的纵坐标范围，如一个数据就显示该点即可，多个数据点就显示3~5度；LEAP机型显示“LEAP机型离地姿态数据图形有误”
        3.进近1000-100ft杆量：对于这个图形和上述内容中的数据，x为x轴上方所有数据点纵坐标最大值，y为x轴下方所有数据点纵坐标最大值的绝对值，z为所有数据点横坐标绝对值的最大值，ws为每个人每种机型生成的“进近1000-100英尺风向风速”图形中的所有数据的径向最大值
        4.落地100ft以下杆量：对于这个图形和上述内容中的数据，x为x轴上方所有数据点纵坐标最大值，y为x轴下方所有数据点纵坐标最大值的绝对值，z为所有数据点横坐标绝对值的最大值，ws为每个人每种机型生成的“100英尺以下风向风速”图形中的所有数据的径向最大值
        5.50ft以下曲线及收油门时机：收油门范围如果只有一个数据点，则显示为该点纵坐标值，如有多个点，则显示所有点纵坐标值的范围
